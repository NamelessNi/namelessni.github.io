redis(remote dictionary sever)远程字典服务
Redis是啥？用Redis官方的话来说就是：

Redis is an open source (BSD licensed), in-memory data structure store, used as a database, cache and message broker.
Redis是一个开源的、基于内存的数据结构存储器，可以用作数据库、缓存和消息中间件。

What??? 这玩意把数据放在内存，还想当数据库使？为什么是“data structure store”，而不是“data store”？还能用作消息中间件？？你这么牛，你咋不上天？

redis基础知识

redis有16个数据库，默认使用第一个数据库。
可以用作数据库，缓存，消息中间件。

基本命令

keys * //查看所有的key
set name light //set key
EXISTS name //判断当前key是否存在
move name 1 //移除当前key
expire name 10 //设置key的过气时间，单位是S
ttl name //查看当前key的剩余时间
type name //查看当前key的类型
append key “hello” //给key后面追加hello

string字符串类型

set key value //设置键值对

get key //通过键获得值

strlen key //获取字符串长度

incr key //当前key+1

decr key //当前key-1

setex //设置过期时间


setnx //不存在再设置 是一个原子性操作

setnx mykey “redis” //如果mykey不存在，创建mykey，成功返回1，不成功返回0


mset //批量设置
mget //批量获取
getset db redis //如果不存在值，则返回nil
get db 会得到redis
getset db mongodb //如果存在值，则获取原来的值，并设置新的值

string类似的使用场景：value可以是字符串和数字
1.计算器；
2.统计多单位的数量；
3.粉丝数
4.对象缓存

list列表类型

所有的list命令基本是l开头
lpush list one //将一个值或多个值，插入到列表头部（左）

rpush list right //将一个值或多个值，插入到列表的尾部（右）
lpop和rpop是移除命令；
lindex 根据下标获取list中的某一个值；

redis事务

redis事务本质：一组命令的集合，一个事务中的所有命令都会被序列化，在事务执行过程中，会按照顺序执行

特点：一次性，顺序性，排他性。
redis事务没有隔离级别的概念；redis单条命令是保存原子性的，但是事务不保证原子性；


redis的事务：开启事务（multi），命令入队（。。。。），执行事务（exec）


discard（取消事务）


命令错误，即代码写错了，事务中所有命令都不会执行。

逻辑性错误，比如1/0；错误命令不执行，其他命令执行。

redis实现乐观锁

悲观锁：很悲观，什么时候都会出问题，所以做什么都加锁。

乐观锁：很乐观，人为什么时候都不会出问题，所以不会上锁，更新数据的时候判断一下，在次期间是否有人修改过这个数据。


使用watch可以当作redis的乐观锁操作，如果这个线程执行之前，数据被修改，则这个线程执行不会成功。
